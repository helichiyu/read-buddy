# TODO: Read Buddy - AI 驱动的书籍阅读记录与推荐工具 - 2026/06/02

## 需求描述

开发一个 AI 驱动的书籍阅读记录与智能推荐工具。用户通过对话式交互录入已读书籍和评价，AI 根据阅读记录推荐新书籍。采用"单机游戏存档"模式，无需注册登录，数据本地存储，支持导出/导入备份。最终打包为 EXE 安装程序，用户安装后通过快捷方式打开独立窗口使用。

### 核心功能

1. **AI 对话录入：** 通过对话式交互，AI 追问用户阅读经历，分三种场景处理：
   - 用户没看过任何书 → 询问兴趣类型 → 推荐新手入门书籍
   - 用户描述了类型但没说书名 → 让用户描述内容 → AI 猜书 → 猜对后评价
   - 用户说出具体书名 → 直接评价
2. **持续追问：** 每次只处理一本书，处理完追问下一本，直到用户想不起来为止
3. **书籍评价：** 星级（1-5 星）+ 文字评价
4. **AI 智能推荐：** 基于所有阅读历史，AI 推荐新的书籍
5. **书籍状态管理：** 书籍有明确的生命周期状态（见下方详细说明）
6. **日常回访：** 每次重新打开后，AI 询问推荐的书看了没有，引导用户继续互动
7. **书籍详情展示：** 通过 Google Books API 获取封面、简介、作者等信息
8. **存档管理：** SQLite 本地存储 + JSON 导出/导入
9. **AI 配置：** 用户只需输入 API Base URL + API Key，自动兼容所有 OpenAI 格式的 API（DeepSeek、Claude、OpenAI、Mistral 等），无需选择供应商
10. **Token 显示：** 每次对话后显示本次消耗 Token 数 + 累计消耗总量
11. **个性化设置：** 用户可自定义 AI 昵称、开场问候语等个人偏好，存档中一并保存和恢复

**侧边栏展示：**
- 只展示"待阅读"状态的推荐书籍
- 每本书显示：封面、书名、作者、推荐原因、书籍简介

**日常回访流程：**
1. 用户重新打开应用
2. AI 问候："欢迎回来！上次推荐了几本书，你看了吗？"
3. 逐本询问推荐书籍的状态：
   - 看了 → 开始评价流程（星级 + 文字评价）→ 存档
   - 还没看 → 继续保留在推荐列表，下次再问
   - 不想看了 → 询问原因，记录后存档，供以后推荐时参考（避免推荐类似书籍，或根据原因调整推荐策略）

### 上下文管理策略

采用"结构化摘要 + 滑动窗口"方案，避免对话历史过长：

- **System Prompt 中注入：**
  - 已读书籍摘要（每本一行：书名、作者、星级、简评）→ 用于推荐逻辑
  - 当前待阅读书籍列表 + 推荐原因 → 用于回访询问
  - 不想看的书籍列表 + 原因 → 避免重复推荐
- **对话历史：** 只保留最近 20 条消息

这样不管用了多久，每次请求的上下文都在可控范围内。

### 使用场景

- **首次使用：** AI 自我介绍 → 询问阅读经历 → 引导录入和评价 → 推荐书籍
- **日常使用：** 打开后加载历史上下文 → AI 问候 → 询问推荐的书看了没 → 引导继续互动
- **随时退出：** 数据自动保存到本地 SQLite，下次打开自动恢复

### 用户界面

- **混合式布局：** 左侧聊天对话区 + 右侧推荐书籍卡片列表
- **苹果风格：** 毛玻璃、圆角、简洁排版、浅色模式
- **纯中文界面**
- **独立窗口：** 不在浏览器中打开，使用原生桌面窗口

### 技术栈

- **后端：** Python + FastAPI + SQLite
- **前端：** 原生 HTML/CSS/JS（苹果风格，不使用重型框架）
- **桌面窗口：** pywebview（将 Web 界面渲染在独立窗口中）
- **AI：** 统一 OpenAI 兼容接口，支持任意供应商
- **书籍数据：** Google Books API（后续可扩展）
- **打包：** PyInstaller → EXE + Inno Setup 安装程序

from openai import AsyncOpenAI

class AIService:
    def __init__(self, base_url: str, api_key: str, model: str):
        self.client = AsyncOpenAI(base_url=base_url, api_key=api_key)
        self.model = model

    async def chat(self, messages: list[dict], system_prompt: str, tools: list[dict] | None = None) -> tuple[object, int]:
        """
        发送对话请求
        返回：(AI 回复 message 对象, 消耗 token 数)
        message 对象包含 .content（文本回复）和 .tool_calls（工具调用）
        """
        kwargs = {
            "model": self.model,
            "messages": [{"role": "system", "content": system_prompt}, *messages],
        }
        if tools:
            kwargs["tools"] = tools
        response = await self.client.chat.completions.create(**kwargs)
        return response.choices[0].message, response.usage.total_tokens

#### 3.3 对话接口逻辑

`POST /api/chat` 核心流程：

1. **构建上下文摘要**（注入 System Prompt）：
   - 从数据库查询已评价书籍 → "已读书籍摘要"
   - 从数据库查询 pending 书籍 → "待阅读列表"（含推荐原因）
   - 从数据库查询 not_interested 书籍 → "不想看列表"（含原因）
2. **加载对话历史**：最近 20 条消息
3. **判断场景**：
   - 无任何记录 → 首次对话（AI 自动发欢迎语）
   - 有 pending 书籍 → 日常回访（AI 询问推荐书看了没）
   - 其他 → 继续上次对话
4. **发送给 AI（带工具定义）** → 获取回复 + 工具调用 + Token 消耗
5. **处理工具调用**（Function Calling）：

   通过 OpenAI 兼容的 Function Calling 机制，AI 会在需要时主动调用预定义的工具函数，后端直接获得结构化数据，无需解析自然语言。

   定义的工具函数：

   ```python
   tools = [
       {
           "type": "function",
           "function": {
               "name": "rate_book",
               "description": "用户对一本书做出了评价",
               "parameters": {
                   "type": "object",
                   "properties": {
                       "book_title": {"type": "string", "description": "书名"},
                       "stars": {"type": "integer", "description": "星级评分 1-5"},
                       "review": {"type": "string", "description": "用户的文字评价"},
                   },
                   "required": ["book_title", "stars"],
               },
           },
       },
       {
           "type": "function",
           "function": {
               "name": "recommend_books",
               "description": "向用户推荐书籍",
               "parameters": {
                   "type": "object",
                   "properties": {
                       "books": {
                           "type": "array",
                           "items": {
                               "type": "object",
                               "properties": {
                                   "title": {"type": "string", "description": "书名"},
                                   "author": {"type": "string", "description": "作者"},
                                   "reason": {"type": "string", "description": "推荐原因"},
                               },
                               "required": ["title", "author", "reason"],
                           },
                       },
                   },
                   "required": ["books"],
               },
           },
       },
       {
           "type": "function",
           "function": {
               "name": "reject_book",
               "description": "用户表示不想看某本推荐的书",
               "parameters": {
                   "type": "object",
                   "properties": {
                       "book_title": {"type": "string", "description": "书名"},
                       "reason": {"type": "string", "description": "不想看的原因"},
                   },
                   "required": ["book_title", "reason"],
               },
           },
       },
   ]
   ```

   后端处理流程：
   - AI 回复中包含 `tool_calls` → 根据 `function.name` 分发处理：
     - `rate_book` → 存入评价 → 调用 Google Books API 获取详情 → 更新书籍状态为 `rated`
     - `recommend_books` → 逐本调用 Google Books API → 存为 `pending` 状态
     - `reject_book` → 更新书籍状态为 `not_interested`，记录原因
   - 将工具调用的结果作为 `tool` 角色消息回传给 AI，让 AI 继续对话
6. **存储**：消息、Token 统计、新增书籍/评价
7. **返回**：AI 回复 + Token 信息 + 新增/变更的书籍数据

#### 3.4 前端对接

- 发送消息 → `POST /api/chat` → 渲染回复 + 更新 Token + 更新侧边栏
- 打开页面 → `GET /api/messages/recent` + `GET /api/books/pending` → 恢复界面
- AI 回复使用打字效果
