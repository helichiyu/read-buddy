#### T1.3 抽公共路径工具 `backend/paths.py`  ✅

4 处 `sys.frozen` 判断逻辑**不同**，工具需区分「资源目录（只读）」与「数据目录（可写）」：

```python
# backend/paths.py
"""路径工具 - 统一开发态与打包态的路径判断"""
import os
import sys
from pathlib import Path


def is_frozen() -> bool:
    """是否运行在 PyInstaller 打包环境"""
    return getattr(sys, "frozen", False)


def resource_dir() -> Path:
    """只读资源目录（打包后为 _MEIPASS，开发态为项目根）"""
    if is_frozen():
        return Path(sys._MEIPASS)
    return Path(__file__).resolve().parent.parent


def data_dir() -> Path:
    """可写数据目录（打包后为 EXE 同级 data/，开发态为项目根 data/）"""
    if is_frozen():
        return Path(os.path.dirname(sys.executable)) / "data"
    return resource_dir() / "data"
```

替换：`app.py`（`FRONTEND_DIR = resource_dir() / "frontend"`）、`database.py`（`DB_DIR`/`DB_PATH` 用 `data_dir()`）、`preferences.py`（`PREF_DIR` 用 `data_dir()`）。

> **权衡**：`main.py` 的 `sys.frozen` 判断处于「设置 sys.path 的引导阶段」，此时 `backend/` 尚未加入 `sys.path`，无法 `import paths`（鸡生蛋）。因此 **`main.py` 保留内联引导**，其余 3 处统一走 `paths.py`，把 4 处重复收敛为「1 处引导 + 1 个公共工具」。

### 阶段二：重构（行为不变）

#### T2.1 `database.py` 应用级单连接  ✅

每个函数原本都 `connect` + `close`。改为启动时建立单连接、复用到关闭：

```python
# database.py
from paths import data_dir

DB_DIR = str(data_dir())
DB_PATH = str(data_dir() / "readbuddy.db")

_db: aiosqlite.Connection | None = None

async def get_db() -> aiosqlite.Connection:
    """获取单例连接（首次调用时建目录、建表）"""
    global _db
    if _db is None:
        os.makedirs(DB_DIR, exist_ok=True)
        _db = await aiosqlite.connect(DB_PATH)
        _db.row_factory = aiosqlite.Row
        await _init_schema(_db)
    return _db

async def close_db() -> None:
    global _db
    if _db is not None:
        await _db.close()
        _db = None
```

- 把 `init_db()` 的建表 `executescript` 抽成内部 `_init_schema(db)`，由 `get_db()` 首次调用触发。
- 去掉全部 22 处业务函数的 `try/finally: await db.close()`。
- `app.py` 增加 `@app.on_event("shutdown")` 调 `db.close_db()`。

> 单用户桌面应用，单连接由 aiosqlite 内部队列串行化执行，足够安全。

### 阶段四：行为变更（单独回归验证）

#### T4.1 `book_service` 内存缓存（#7）  ✅

模块级缓存，键用 `search()` 的完整归一化 query（大小写折叠），只缓存命中结果（失败不缓存）。豆瓣请求仍使用 `simple_query`，避免带空格复合查询失败；缓存不用 `simple_query`，避免英文书名同首词串结果：

```python
# book_service.py
_search_cache: dict[str, dict] = {}  # 进程内，重启失效

async def search(query: str) -> dict | None:
    normalized_query = " ".join(query.split())
    cache_key = normalized_query.casefold()
    simple_query = normalized_query.split()[0] if " " in normalized_query else normalized_query
    if cache_key in _search_cache:
        return _search_cache[cache_key]
    result = await _search_douban(simple_query)
    if not result:
        result = await _search_google(normalized_query)
    if result:
        _search_cache[cache_key] = result
    return result
```

- 命中场景：`accept_book` 后再 `rate_book` 同一本书不重复查外部 API。
- 验证（mock）：相同书名命中缓存、不同书名重新查询、同首词不同英文书名不串缓存。
